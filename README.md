# gestion-deuvre-social
نظام إدارة الاقتطاعات والمنح الاجتماعية
## 📋 ملخص التطبيق والتغييرات الكاملة

### 1. نظرة عامة على النظام
هو نظام **إدارة الاقتطاعات والمنح الاجتماعية** خاص بمركز التكوين المهني (لجنة الخدمات الاجتماعية). يعمل بـ **PHP 8 + SQLite** عبر **PDO**، ويهدف إلى رقمنة وإدارة:
- الموظفين (دائمون/متعاقدون) مع أرقام حساباتهم.
- الاقتطاعات (سلف، قروض، اقتطاعات شهرية عادية).
- المنح الاجتماعية (ثابتة/نسبية مئوية مع حدود قصوى).
- الأقساط الشهرية (إنشاء، تأجيل، تسديد).
- المحاضر الشهرية للجنة (تحرير، طباعة).
- الميزانية الاجتماعية (متابعة الصرف والاسترجاعات).

---

### 2. التعديلات الجوهرية التي طُبقت

#### أ. المنح الاجتماعية (Grants)
- **أنواع المنح**:
  - **ثابتة** (`fixed`): مبلغ محدد مسبقاً.
  - **نسبة مئوية** (`percentage`): تحسب من قيمة الفاتورة، مع إمكانية تحديد حد أقصى (مثل 50% بحد 25000 دج).
- **عرض القيمة الصحيحة**: تم تعديل الاستعلامات لاستخدام `eg.amount` (القيمة المحسوبة) بدلاً من `g.amount` (القيمة الثابتة)، مما يُظهر المنح النسبية بقيمها الفعلية في كل من قائمة المنح والمحاضر.
- **آلية التحديث والمزامنة**:
  - زر **"تحديث"**: يُحدّث المنحة الموزعة لتتطابق مع قيمة المنحة الأم (إذا كانت `current_amount > 0`).
  - زر **"إعادة حساب"**: يُعيد حساب المنح النسبية بناءً على قيمة الفاتورة المخزنة (`invoice_amount`).
  - زر **"تعيين القيمة"**: للمنح التي `current_amount = 0` (مثل وجبات المطعم)، يُحدّث المنحة الأم والموزعة معاً.
- **ربط الميزانية**: تم إضافة تسجيل تلقائي في `budget_transactions` عند توزيع أي منحة (`type = 'grant'`, `is_deduct = 1`) مع تحديث `social_budget.remaining_budget`.

#### ب. الاقتطاعات والأقساط (Deductions & Installments)
- **تاريخ الصرف للسلف**: تمت إضافة عمود `grant_date` في جدول `deductions`، وتعديل واجهات الإضافة والتعديل لتضمينه، وعرضه في المحاضر باستخدام `safeFormatDate()` و `COALESCE`.
- **تأجيل الأقساط الفردية**:
  - تم إنشاء صفحة جديدة `postpone_installment.php`.
  - إضافة أعمدة (`is_postponed`, `postponed_from_month`) في `monthly_installments`.
  - إنشاء جدول `installment_postponements` لتسجيل كل عملية تأجيل (القسط الأصلي، الشهر الجديد، السبب، المستخدم).
  - **منطق التأجيل**: عند تأجيل قسط، يتم:
    1. تعليم القسط الحالي كمؤجل (`is_postponed = 1`).
    2. إنشاء قسط جديد في **أول شهر فارغ** بعد آخر قسط موجود (بدلاً من نهاية المدة مباشرة، لتجنب الفجوات).
    3. تمديد تاريخ نهاية الاقتطاع (`end_date`) تلقائياً.
    4. إمكانية **إلغاء التأجيل** لعكس العملية بالكامل.
- **تسديد الأقساط الشهرية**:
  - تمت إضافة زر **"تسديد"** و **"تسديد الكل"** في التقرير الشهري (`monthly.php`).
  - **لا يتم تغيير `total_months` أو `end_date`** عند تسديد قسط عادي (يبقى تاريخ النهاية ثابتاً كما في الواقع).
  - إذا كان القسط يخص **سلفة** (`is_loan = 1`)، يتم **إعادة المبلغ إلى الميزانية** (`social_budget`) وتسجيله كـ **استرجاع** في `budget_transactions` (`type = 'installment'`, `is_deduct = 0`).
  - استخدام **مودال تأكيد** (نافذة منبثقة) لتحسين تجربة المستخدم بدلاً من `confirm()`.

#### ج. المحاضر الشهرية (Meeting Minutes)
- **عرض المنح المحسوبة**: تعديل الاستعلامات لاستخدام `eg.amount` بدلاً من `g.amount`.
- **عرض تاريخ السلف**: استخدام `COALESCE(grant_date, created_at, start_date)` وإضافة دالة `safeFormatDate()` لتجنب أخطاء `strtotime(null)` وظهور `01/01/1970`.
- **أرقام الحسابات**: إضافة عمود `account_number` في جدول `employees` وعرضه أسفل اسم كل موظف في جداول المحضر (المنح، الوجبات، السلف، أعضاء اللجنة).
- **تحسين الطباعة**: تعديل تنسيق `@media print` لتحسين مظهر المحضر المطبوع (حجم الخط `11pt`، `padding` معتدل، خط `line-height` مناسب).

#### د. لوحة التحكم والتقارير المالية
- **تحديث الميزانية**: أصبحت كل عملية مالية تُسجل في `budget_transactions`:
  - **المنح**: `type='grant'`, `is_deduct=1` (مبلغ المنحة الفعلي).
  - **السلف**: `type='loan'`, `is_deduct=1` (المبلغ الكلي للسلفة).
  - **استرجاعات السلف (عند تسديد قسط)**: `type='installment'`, `is_deduct=0`.
- **تصحيح البيانات الخاطئة**: تقديم استعلامات SQL لتصحيح المنح غير المسجلة في الميزانية أو المسجلة بقيمة خاطئة (مثل منح قفة رمضان التي كانت تُسجل بـ 5000 بدلاً من 7000).

### 3. التعديلات في قاعدة البيانات

| الجدول | الأعمدة/التعديلات المضافة |
|--------|---------------------------|
| `grants` | `calculation_type`, `percentage_value`, `max_amount`, `created_at`, `updated_at` |
| `employees` | `account_number` |
| `monthly_installments` | `is_postponed`, `postponed_from_month` |
| `installment_postponements` | **(جدول جديد)** `id`, `installment_id`, `deduction_id`, `employee_id`, `original_month`, `new_month`, `reason`, `postponed_by`, `postponed_at` |
| `employee_grants` | `amount` (المبلغ الفعلي المحسوب), `invoice_amount` (قيمة الفاتورة) |
| `deductions` | `grant_date` (تاريخ صرف السلفة), `included_in_minute_id` (ربطها بالمحضر) |
| `meeting_minutes` | `show_djezzy`, `show_cheques`, `show_honorees` وغيرها من خيارات العرض |

| `deductions/postpone_installment.php` | **(جديد)** إدارة تأجيل الأقساط الفردية. |

| `grants/assign.php` | إضافة تسجيل المنحة في الميزانية وإدخال قيمة الفاتورة. |
| `grants/employee_list.php` | تحسين أزرار "تحديث"، "إعادة حساب"، وإضافة مودال الحذف. |
| `deductions/view.php` | عرض جميع الأقساط (مدفوعة/غير مدفوعة) مع تمييز الحالات حسب التاريخ. |

| `dashboard.php` & `report.php` | تحسين حساب الميزانية بناءً على `budget_transactions`. |

### 5. ملخص المشاكل التي تم حلها

| المشكلة | الحل |
|---------|------|
| المنح النسبية تظهر بقيمة 0 في المحاضر | استخدام `eg.amount` بدلاً من `g.amount`. |
| عدم تحديث الميزانية عند تسديد الأقساط | إضافة سجل في `budget_transactions` مع `is_deduct=0`. |
| السلف لا تظهر في المحضر | إضافة `grant_date` واستخدام `COALESCE`. |
| الأقساط المسددة تختفي من `view.php` | عرض جميع الأقساط (مدفوعة وغير مدفوعة). |
| أخطاء `strtotime(null)` | إنشاء دالة `safeFormatDate()`. |
| جدول `installment_postponements` مفقود | إنشاء الجدول وإضافة الأعمدة المطلوبة. |
| صعوبة تسديد أقساط متعددة | إضافة زر "تسديد الكل" في التقرير الشهري. |

---

### ✅ حالة النظام الحالية
- أصبح النظام يدعم **المنح المعقدة** (نسب مئوية، حدود قصوى) مع مزامنة كاملة مع الميزانية.
- أصبحت **الأقساط** تُدار بمرونة (تأجيل فردي، تسديد جماعي، عرض تاريخي كامل).
- أصبحت **المحاضر** تعرض البيانات الدقيقة (منح محسوبة، تواريخ صرف، أرقام حسابات) مع إمكانية طباعة احترافية.
- أصبحت **الميزانية** مُحدثة تلقائياً مع كل عملية (منح، سلف، تسديد، استرجاع).
